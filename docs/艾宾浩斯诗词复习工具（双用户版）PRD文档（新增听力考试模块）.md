# 二、用户需求（User Requirement）

## 2\.1 核心需求（新增听力考试相关）

- 听力考试功能：新增独立听力考试模块，支持用户选择诗词进行听力练习，通过播放诗词音频，完成答题，强化听读记忆。

- 音频控制：支持音频播放、暂停、重播，可调节音量，适配不同环境下的听力练习需求。

- 听力答题模式：结合诗词特点，设计贴合复习场景的听力答题模式，与原有答题模式互补，数据同步计入个人学习统计。

- 双用户数据隔离：听力考试记录、答题情况，分别计入当前用户的学习数据，与另一位用户互不干扰，支持重置个人听力相关记录（随个人全部学习记录一同重置）。

- 适配原有逻辑：听力考试完成后，同步更新该诗词的复习进度，纳入艾宾浩斯复习周期，不破坏原有复习机制。

## 2\.2 原有核心需求（保留）

- 双用户独立使用：同学A、同学B共用一个网页，可一键切换身份，数据完全隔离，互不干扰，支持各自重置个人学习数据。

- 艾宾浩斯复习机制：严格按照标准艾宾浩斯遗忘周期（0天、1天、2天、4天、7天、15天、30天）生成复习计划，首次学习后自动生成后续所有复习日期，漏复习自动顺延，完成全部周期后标记为“已掌握”，不再提醒。

- 多模式答题练习：支持4种复习模式自由切换，手动输入作答，提供“查看答案”功能，满足不同复习场景需求。

- 诗词题库管理：内置380\+首完整诗词，支持按分类筛选复习范围，可手动新增、编辑、删除诗词。

- 数据统计与可视化：提供首页今日待办快捷入口、月历视图（标注待复习任务）、个人学习统计、双用户进度对比，直观展示学习状态。

- 本地数据存储：纯本地永久保存数据，关闭浏览器、刷新页面后数据不丢失，无需登录、无需联网（首次打开后）。

## 2\.3 非核心需求（已实现，新增听力相关适配）

- 界面简洁：采用极简校园清爽风，浅底色、清晰分区，护眼适配，无广告、无复杂动画，专注复习功能；听力模块界面与整体风格统一。

- 隐私保护：双用户仅能查看对方整体学习统计数据，无法查看单首诗词的学习细节（含听力答题细节），保护个人学习隐私。

- 操作便捷：听力考试流程简单，仅需“播放音频\-作答\-查看答案\-完成”，降低使用门槛，与原有答题操作逻辑保持一致。

## 2\.4 排除需求（不实现，新增听力相关补充）

- 移动端适配：仅支持电脑端浏览器，不做手机端、平板端适配，听力播放功能仅适配电脑端音频输出。

- 自动判分功能：仅支持手动输入作答、自行核对答案，听力考试亦不实现AI智能批改、对错判断。

- 提醒功能：不提供弹窗、声音等提醒，仅通过日历、今日待办列表展示待复习任务，听力考试不额外增加提醒。

- 云端存储：不支持数据同步、云端备份，仅本地存储，听力音频文件内置，不支持云端上传/下载。

- 排名与奖惩：双用户进度仅做静态对比，不设置排名、积分、奖惩机制，听力考试成绩不参与排名。

- 手动解除“已掌握”：诗词完成全部复习周期后自动标记为“已掌握”，暂不支持手动解除该状态、重新复习，听力考试亦遵循此规则。

- 自定义音频：不支持用户上传自定义诗词音频，仅使用工具内置的诗词音频文件。

# 三、产品功能模块（Functional Module）

## 3\.1 整体结构（新增听力考试模块后）

工具整体分为5个核心页面，顶部固定导航栏（支持页面切换）、用户切换按钮，页面采用卡片式布局，简洁清晰，核心模块如下：

顶部导航栏：首页 → 我的复习计划 → 诗词库 → 听力考试 → 个人中心

用户切换：一键切换“同学A”“同学B”，无需密码，切换后自动加载当前用户数据（含听力考试相关记录）。

## 3\.2 核心模块详细说明（新增听力考试模块，其余模块保留并适配）

### 3\.2\.1 首页模块（适配听力考试）

核心作用：展示今日待办、双用户进度对比，提供快捷复习入口，直观呈现整体学习状态，新增听力考试快捷入口。

- 今日待办快捷入口：显示当前用户今日需复习诗词数量，提供“前往复习”按钮，点击直接跳转至“我的复习计划”页面。

- 听力考试快捷入口：新增“听力练习”按钮，点击直接跳转至“听力考试”页面，方便用户快速进入听力练习。

- 双用户进度对比面板：同屏展示同学A、同学B的4项核心数据，无排名、无奖惩，仅做静态对比：
        

    - 已学诗词数：累计学习的诗词总数量

    - 待复习数：当前处于复习周期内、未完成当日复习的诗词数量

    - 已掌握数：完成全部艾宾浩斯复习周期、标记为“已掌握”的诗词数量

    - 听力练习次数：累计完成的听力考试次数（新增）

### 3\.2\.2 我的复习计划模块（适配听力考试）

核心作用：展示复习日历、今日待复习列表，提供复习入口，实现复习任务的集中管理，听力考试完成后同步更新复习状态。

- 复习日历：月历视图，采用不同颜色标注日期状态，直观展示待复习任务：
        

    - 今日：突出显示，标注今日待复习诗词数量

    - 有任务日期：标注该日期有待复习诗词

    - 普通日期：无特殊标注

- 今日待复习列表：展示当前用户今日需复习的所有诗词，每首诗词提供“开始复习”“听力练习”两个按钮（新增听力练习按钮）：
        

    - 开始复习：点击直接进入原有答题弹窗，完成文字答题复习。

    - 听力练习：点击直接进入听力考试弹窗，通过音频播放完成听力复习。

### 3\.2\.3 诗词库模块（适配听力考试）

核心作用：展示全量诗词、提供分类筛选、诗词管理功能，是用户新增学习诗词的入口，新增听力练习入口。

- 分类筛选：预设分类标签（唐诗、宋词、乐府、古诗、古文等），支持“全部”“单个分类”筛选，点击分类标签可快速切换展示对应分类的诗词，筛选状态直观显示（选中分类标签高亮）。

- 诗词列表：展示当前筛选条件下的所有诗词，每首诗词显示“题目、作者、全文”，并提供4个操作按钮（新增听力练习按钮）：
        

    - 学习/复习：首次点击，将该诗词加入当前用户学习计划，自动生成艾宾浩斯复习周期；非首次点击，直接进入答题弹窗进行文字复习。

    - 听力练习：点击直接进入听力考试弹窗，对该诗词进行听力练习，完成后同步更新复习进度（新增）。

    - 编辑：可修改诗词的题目、作者、分类、全文，修改后实时保存至本地（不涉及音频修改，音频与诗词一一对应，不可编辑）。

    - 删除：删除该诗词，删除前弹出确认提示，删除后同步更新诗词列表（不可恢复），同时删除该诗词对应的听力相关记录。

- 新增诗词：提供“新增诗词”按钮，点击后通过弹窗输入“题目、作者、分类、全文”，确认后新增至诗词库，支持后续学习、编辑、删除；新增诗词暂不支持自定义上传音频，默认无音频，听力练习按钮置灰（不可点击）。

### 3\.2\.4 个人中心模块（适配听力考试）

核心作用：展示当前用户的学习统计数据、提供个人数据重置功能，实现个人学习状态的查看与管理，新增听力相关统计。

- 我的学习数据：展示当前用户的5项核心统计指标，实时更新（新增1项听力相关）：
        

    - 已学习：累计学习的诗词总数量

    - 待复习：当前处于复习周期内、未完成当日复习的诗词数量

    - 已掌握：完成全部艾宾浩斯复习周期、标记为“已掌握”的诗词数量

    - 听力练习次数：累计完成的听力考试次数（新增）

    - 听力练习诗词数：累计进行过听力练习的诗词总数量（新增）

- 数据管理：提供“重置我的所有学习记录”按钮，点击后弹出确认提示（提示“不可恢复”），确认后清空当前用户的所有学习数据（学习列表、复习记录、已掌握列表、听力练习记录），不影响另一位用户的数据。

### 3\.2\.5 答题弹窗模块（核心交互，保留）

核心作用：实现多模式答题练习，完成复习标记，是复习流程的核心交互载体，弹出后覆盖当前页面，关闭后返回原页面并更新数据。

- 模式切换：提供4种复习模式，可自由切换，切换后实时更新题目展示：
        

    - 模式1：题目背全文 → 展示题目\+作者，用户手动输入全文

    - 模式2：上句填下句 → 展示诗词上句，用户手动输入下句

    - 模式3：下句填上句 → 展示诗词下句，用户手动输入上句

    - 模式4：诗句猜题目 → 展示诗词某一句，用户手动输入题目\+作者

- 答题区域：文本框输入，支持多行输入，适配诗词换行需求。

- 操作按钮：
        

    - 查看答案：点击后展示该诗词的完整信息（题目、作者、全文），方便用户自查核对。

    - 完成复习：点击后标记该次复习完成，自动计算并更新下一次复习日期；若已完成全部复习周期，标记为“已掌握”，从待复习列表中移除，关闭弹窗并更新所有相关数据。

    - 关闭：关闭答题弹窗，不记录本次复习，返回原页面。

### 3\.2\.6 听力考试模块（新增核心模块）

核心作用：实现诗词听力练习，通过音频播放强化用户听读记忆，完成听力复习后同步更新诗词复习进度，与原有复习机制联动，弹出式交互，不新增独立答题页面（与原有答题弹窗风格统一）。

- 入口场景：3个核心入口，方便用户快速进入：

    - 首页：“听力练习”快捷按钮，点击后进入听力考试选择界面（选择要练习的诗词）。

    - 我的复习计划：今日待复习列表中，每首诗词的“听力练习”按钮，点击直接进入该诗词的听力考试弹窗。

    - 诗词库：每首诗词的“听力练习”按钮，点击直接进入该诗词的听力考试弹窗（无音频的诗词，按钮置灰不可点击）。

- 听力考试选择界面（首页入口跳转后）：
        

    - 分类筛选：与诗词库分类一致，支持按分类筛选可进行听力练习的诗词（仅显示有内置音频的诗词）。

    - 诗词列表：展示当前筛选条件下，有内置音频的诗词，每首诗词显示“题目、作者”，提供“开始听力练习”按钮，点击进入该诗词的听力考试弹窗。

- 听力考试弹窗（核心交互）：
        

    - 诗词信息展示：顶部显示当前听力练习的诗词“题目、作者”，不显示全文（避免提前查看，贴合听力场景）。

    - 音频控制区域：
                

        - 播放/暂停按钮：点击播放音频，播放过程中显示“暂停”按钮，暂停后显示“播放”按钮；音频播放完毕后，自动重置为“播放”状态，支持重复播放。

        - 重播按钮：无论音频是否播放完毕，均可点击重播，重新播放当前诗词音频。

        - 音量调节：提供音量滑块，支持0\-100%调节，适配不同环境下的听力需求。

        - 音频提示：音频播放时，显示“正在播放”提示；无音频的诗词（若误触发），显示“暂无音频”提示，且无法播放。

    - 听力答题模式（固定1种，贴合听力场景）：
                

        - 模式：听音频填全文 → 播放诗词完整音频（可重复播放），用户手动输入诗词全文，适配诗词换行需求，支持多行输入。

        - 答题提示：可提示“播放音频后，输入诗词全文”，引导用户操作。

    - 操作按钮（与原有答题弹窗逻辑一致，保持操作统一）：
                

        - 查看答案：点击后展示该诗词的完整信息（题目、作者、全文），方便用户自查核对，同时暂停音频播放。

        - 完成听力复习：点击后标记该次听力复习完成，同步更新该诗词的复习记录（计入艾宾浩斯复习周期，与文字答题复习同等效力），更新个人听力统计数据；若已完成全部复习周期，标记为“已掌握”，从待复习列表中移除，关闭弹窗并更新所有相关数据。

        - 关闭：关闭听力考试弹窗，不记录本次听力练习，停止音频播放，返回原页面。

- 音频资源说明：工具内置380\+首诗词对应的音频文件，与诗词一一对应（通过诗词索引关联），音频格式为MP3，确保浏览器兼容播放；新增诗词默认无音频，听力练习按钮置灰，不可点击。

# 四、核心业务逻辑（Business Logic）

## 4\.1 双用户数据隔离逻辑（新增听力相关数据）

- 本地存储区分：通过localStorage分别存储“userA”“userB”的数据，数据结构完全一致，互不关联，新增听力相关字段。

- 数据结构（用户数据，新增听力相关字段）：


    - learn：数组，存储当前用户已学习的诗词索引（对应诗词库中的索引）。

    - done：数组，存储当前用户已掌握的诗词索引（完成全部复习周期）。

    - log：对象，存储每首诗词的复习记录，key为诗词索引，value包含3个字段：
                

        - t：首次学习时间（时间戳）

        - c：已完成复习次数（文字答题\+听力答题，累计计算）

        - next：下一次复习时间（时间戳）

    - listeningLog：对象，存储每首诗词的听力练习记录（新增），key为诗词索引，value包含2个字段：
                

        - count：该诗词的听力练习次数

        - lastTime：最后一次听力练习时间（时间戳）

    - listeningTotal：数字，当前用户累计完成的听力练习总次数（新增）。

- 切换逻辑：点击“切换A/B”按钮，切换当前用户标识（user=A/B），同步更新页面所有数据（统计、待复习、诗词学习状态、听力相关统计），无需重新加载页面；切换时，若当前有听力音频播放，自动停止播放。

## 4\.2 艾宾浩斯复习周期逻辑（适配听力考试）

- 标准周期：固定采用7个复习间隔，从首次学习当日开始计算：0天（首次学习）→ 1天 → 2天 → 4天 → 7天 → 15天 → 30天。

- 首次学习：用户点击“学习/复习”或“听力练习”按钮，若该诗词未在当前用户学习列表中，自动加入learn数组，记录首次学习时间（t），设置已复习次数（c=0），计算并设置下一次复习时间（next=首次学习时间\+0天，即当日）。

- 复习完成更新：无论是文字答题复习（点击“完成复习”），还是听力答题复习（点击“完成听力复习”），均计入已复习次数（c），c\+1；根据当前c值计算下一次复习时间（next=首次学习时间\+对应周期天数）；若c≥7（完成全部周期），将该诗词索引加入done数组，next设为null，不再提醒复习。

- 漏复习顺延：计算下一次复习时间时，若该时间早于当前日期（即漏复习），自动将next设为当前日期，实现顺延，确保用户不会错过复习，且无需重新开始周期；听力复习完成后，同样遵循此逻辑。

## 4\.3 诗词库数据逻辑（适配听力考试）

- 初始化：工具首次打开时，自动将380\+首内置诗词（包含题目、作者、分类、全文）存入localStorage的“poems”字段，同时存入每首诗词对应的音频路径（新增audio字段），作为初始题库。

- 新增/编辑/删除：
        

    - 新增：用户新增诗词时，仅录入题目、作者、分类、全文，audio字段设为null（无音频），听力练习按钮置灰。

    - 编辑：可修改诗词的题目、作者、分类、全文，无法修改audio字段（音频内置，不可编辑）。

    - 删除：删除该诗词时，同步删除该诗词对应的listeningLog记录，更新listeningTotal统计数据。

- 分类逻辑：诗词分类由用户在新增/编辑时输入，筛选时根据分类字段精确匹配，默认显示“全部”诗词；听力考试选择界面的筛选，仅显示audio字段不为null的诗词。

## 4\.4 数据统计逻辑（新增听力相关统计）

- 实时更新：所有统计数据（原有\+新增听力相关）均实时计算，每次完成复习、新增/删除诗词、切换用户、完成听力练习后，自动重新计算并更新展示。

- 待复习数量计算：筛选当前用户learn数组中，未在done数组中，且next时间在当前日期（00:00\-24:00）内的诗词数量（与听力练习无关，仅看复习周期）。

- 双用户对比：分别读取userA、userB的数据，计算各自的统计指标（含听力相关），同屏展示，互不干扰。

- 听力相关统计计算：


    - 听力练习次数：当前用户listeningTotal字段的值（累计所有诗词的听力练习次数）。

    - 听力练习诗词数：当前用户listeningLog对象中，key的数量（即进行过听力练习的诗词总数）。

## 4\.5 听力考试核心逻辑（新增）

- 音频关联逻辑：诗词库中每首诗词的audio字段存储音频路径，听力考试弹窗根据当前诗词索引，读取对应的audio路径，通过HTML5 Audio API实现播放、暂停、重播、音量调节。

- 听力练习记录逻辑：
        

    - 首次进行某诗词的听力练习：在listeningLog中新增该诗词索引的记录，count设为1，lastTime设为当前时间戳；listeningTotal\+1。

    - 非首次进行某诗词的听力练习：更新该诗词索引对应的listeningLog记录，count\+1，lastTime更新为当前时间戳；listeningTotal\+1。

    - 未完成听力练习（点击“关闭”）：不更新listeningLog和listeningTotal，不更新该诗词的复习记录（c不增加）。

- 音频异常处理：若音频路径错误、浏览器不支持音频播放，显示“音频无法播放”提示，用户可选择“查看答案”或“关闭”弹窗，不影响其他功能使用。

- 听力与复习联动：完成听力复习（点击“完成听力复习”），等同于完成一次正常复习，同步更新该诗词的log记录（c\+1）、next时间，与文字答题复习效力一致，共同推进艾宾浩斯复习周期。
