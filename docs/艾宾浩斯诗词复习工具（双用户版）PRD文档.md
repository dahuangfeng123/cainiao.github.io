# 二、用户需求（User Requirement）

## 2\.1 核心需求

- 双用户独立使用：同学A、同学B共用一个网页，可一键切换身份，数据完全隔离，互不干扰，支持各自重置个人学习数据。

- 艾宾浩斯复习机制：严格按照标准艾宾浩斯遗忘周期（0天、1天、2天、4天、7天、15天、30天）生成复习计划，首次学习后自动生成后续所有复习日期，漏复习自动顺延，完成全部周期后标记为“已掌握”，不再提醒。

- 多模式答题练习：支持4种复习模式自由切换，手动输入作答，提供“查看答案”功能，满足不同复习场景需求。

- 诗词题库管理：内置380\+首完整诗词，支持按分类筛选复习范围，可手动新增、编辑、删除诗词。

- 数据统计与可视化：提供首页今日待办快捷入口、月历视图（标注待复习任务）、个人学习统计、双用户进度对比，直观展示学习状态。

- 本地数据存储：纯本地永久保存数据，关闭浏览器、刷新页面后数据不丢失，无需登录、无需联网（首次打开后）。

## 2\.2 非核心需求（已实现）

- 界面简洁：采用极简校园清爽风，浅底色、清晰分区，护眼适配，无广告、无复杂动画，专注复习功能。

- 隐私保护：双用户仅能查看对方整体学习统计数据，无法查看单首诗词的学习细节，保护个人学习隐私。

- 操作便捷：首页提供今日待办快捷入口，复习流程简单，仅需“作答\-查看答案\-完成标记”三步，降低使用门槛。

## 2\.3 排除需求（不实现）

- 移动端适配：仅支持电脑端浏览器，不做手机端、平板端适配。

- 自动判分功能：仅支持手动输入作答、自行核对答案，不实现AI智能批改、对错判断。

- 提醒功能：不提供弹窗、声音等提醒，仅通过日历、今日待办列表展示待复习任务。

- 云端存储：不支持数据同步、云端备份，仅本地存储。

- 排名与奖惩：双用户进度仅做静态对比，不设置排名、积分、奖惩机制。

- 手动解除“已掌握”：诗词完成全部复习周期后自动标记为“已掌握”，暂不支持手动解除该状态、重新复习。

# 三、产品功能模块（Functional Module）

## 3\.1 整体结构

工具整体分为4个核心页面，顶部固定导航栏（支持页面切换）、用户切换按钮，页面采用卡片式布局，简洁清晰，核心模块如下：

顶部导航栏：首页 → 我的复习计划 → 诗词库 → 个人中心

用户切换：一键切换“同学A”“同学B”，无需密码，切换后自动加载当前用户数据。

## 3\.2 核心模块详细说明

### 3\.2\.1 首页模块

核心作用：展示今日待办、双用户进度对比，提供快捷复习入口，直观呈现整体学习状态。

- 今日待办快捷入口：显示当前用户今日需复习诗词数量，提供“前往复习”按钮，点击直接跳转至“我的复习计划”页面。

- 双用户进度对比面板：同屏展示同学A、同学B的3项核心数据，无排名、无奖惩，仅做静态对比：


    - 已学诗词数：累计学习的诗词总数量

    - 待复习数：当前处于复习周期内、未完成当日复习的诗词数量

    - 已掌握数：完成全部艾宾浩斯复习周期、标记为“已掌握”的诗词数量

### 3\.2\.2 我的复习计划模块

核心作用：展示复习日历、今日待复习列表，提供复习入口，实现复习任务的集中管理。

- 复习日历：月历视图，采用不同颜色标注日期状态，直观展示待复习任务：
        

    - 今日：突出显示，标注今日待复习诗词数量

    - 有任务日期：标注该日期有待复习诗词

    - 普通日期：无特殊标注

- 今日待复习列表：展示当前用户今日需复习的所有诗词，每首诗词提供“开始复习”按钮，点击直接进入答题弹窗，完成复习后自动更新状态，从列表中移除。

### 3\.2\.3 诗词库模块

核心作用：展示全量诗词、提供分类筛选、诗词管理功能，是用户新增学习诗词的入口。

- 分类筛选：预设分类标签（唐诗、宋词、乐府、古诗、古文等），支持“全部”“单个分类”筛选，点击分类标签可快速切换展示对应分类的诗词，筛选状态直观显示（选中分类标签高亮）。

- 诗词列表：展示当前筛选条件下的所有诗词，每首诗词显示“题目、作者、全文”，并提供3个操作按钮：
        

    - 学习/复习：首次点击，将该诗词加入当前用户学习计划，自动生成艾宾浩斯复习周期；非首次点击，直接进入答题弹窗进行复习。

    - 编辑：可修改诗词的题目、作者、分类、全文，修改后实时保存至本地。

    - 删除：删除该诗词，删除前弹出确认提示，删除后同步更新诗词列表（不可恢复）。

- 新增诗词：提供“新增诗词”按钮，点击后通过弹窗输入“题目、作者、分类、全文”，确认后新增至诗词库，支持后续学习、编辑、删除。

### 3\.2\.4 个人中心模块

核心作用：展示当前用户的学习统计数据、提供个人数据重置功能，实现个人学习状态的查看与管理。

- 我的学习数据：展示当前用户的4项核心统计指标，实时更新：
        

    - 已学习：累计学习的诗词总数量

    - 待复习：当前处于复习周期内、未完成当日复习的诗词数量

    - 已掌握：完成全部艾宾浩斯复习周期、标记为“已掌握”的诗词数量

- 数据管理：提供“重置我的所有学习记录”按钮，点击后弹出确认提示（提示“不可恢复”），确认后清空当前用户的所有学习数据（学习列表、复习记录、已掌握列表），不影响另一位用户的数据。

### 3\.2\.5 答题弹窗模块（核心交互）

核心作用：实现多模式答题练习，完成复习标记，是复习流程的核心交互载体，弹出后覆盖当前页面，关闭后返回原页面并更新数据。

- 模式切换：提供4种复习模式，可自由切换，切换后实时更新题目展示：
        

    - 模式1：题目背全文 → 展示题目\+作者，用户手动输入全文

    - 模式2：上句填下句 → 展示诗词上句，用户手动输入下句

    - 模式3：下句填上句 → 展示诗词下句，用户手动输入上句

    - 模式4：诗句猜题目 → 展示诗词某一句，用户手动输入题目\+作者

- 答题区域：文本框输入，支持多行输入，适配诗词换行需求。

- 操作按钮：
        

    - 查看答案：点击后展示该诗词的完整信息（题目、作者、全文），方便用户自查核对。

    - 完成复习：点击后标记该次复习完成，自动计算并更新下一次复习日期；若已完成全部复习周期，标记为“已掌握”，从待复习列表中移除，关闭弹窗并更新所有相关数据。

    - 关闭：关闭答题弹窗，不记录本次复习，返回原页面。

# 四、核心业务逻辑（Business Logic）

## 4\.1 双用户数据隔离逻辑

- 本地存储区分：通过localStorage分别存储“userA”“userB”的数据，数据结构完全一致，互不关联。

- 数据结构（用户数据）：
        

    - learn：数组，存储当前用户已学习的诗词索引（对应诗词库中的索引）。

    - done：数组，存储当前用户已掌握的诗词索引（完成全部复习周期）。

    - log：对象，存储每首诗词的复习记录，key为诗词索引，value包含3个字段：
                

        - t：首次学习时间（时间戳）

        - c：已完成复习次数

        - next：下一次复习时间（时间戳）

- 切换逻辑：点击“切换A/B”按钮，切换当前用户标识（user=A/B），同步更新页面所有数据（统计、待复习、诗词学习状态），无需重新加载页面。

## 4\.2 艾宾浩斯复习周期逻辑

- 标准周期：固定采用7个复习间隔，从首次学习当日开始计算：0天（首次学习）→ 1天 → 2天 → 4天 → 7天 → 15天 → 30天。

- 首次学习：用户点击“学习/复习”按钮，若该诗词未在当前用户学习列表中，自动加入learn数组，记录首次学习时间（t），设置已复习次数（c=0），计算并设置下一次复习时间（next=首次学习时间\+0天，即当日）。

- 复习完成更新：每次点击“完成复习”，已复习次数（c）\+1，根据当前c值计算下一次复习时间（next=首次学习时间\+对应周期天数）；若c≥7（完成全部周期），将该诗词索引加入done数组，next设为null，不再提醒复习。

- 漏复习顺延：计算下一次复习时间时，若该时间早于当前日期（即漏复习），自动将next设为当前日期，实现顺延，确保用户不会错过复习，且无需重新开始周期。

## 4\.3 诗词库数据逻辑

- 初始化：工具首次打开时，自动将380\+首内置诗词（包含题目、作者、分类、全文）存入localStorage的“poems”字段，作为初始题库。

- 新增/编辑/删除：所有操作均直接修改localStorage的“poems”字段，实时同步诗词列表展示，修改后数据永久保存，关闭浏览器不丢失。

- 分类逻辑：诗词分类由用户在新增/编辑时输入，筛选时根据分类字段精确匹配，默认显示“全部”诗词。

## 4\.4 数据统计逻辑

- 实时更新：所有统计数据（已学、待复习、已掌握）均实时计算，每次完成复习、新增/删除诗词、切换用户后，自动重新计算并更新展示。

- 待复习数量计算：筛选当前用户learn数组中，未在done数组中，且next时间在当前日期（00:00\-24:00）内的诗词数量。

- 双用户对比：分别读取userA、userB的数据，计算各自的统计指标，同屏展示，互不干扰。

# 七、异常场景处理（Exception Handling）

|异常场景|处理方式|
|---|---|
|localStorage不可用（浏览器禁用）|工具无法正常使用，弹出提示“请启用浏览器本地存储功能，否则无法保存数据”。|
|新增诗词时，未输入题目/全文|不执行新增操作，无提示（极简设计，默认用户输入完整信息后点击确认）。|
|删除诗词/重置数据时，误操作|弹出确认提示“确定删除？”“确定清空？”，点击“确认”才执行操作，点击“取消”取消操作。|
|诗词全文为空/格式异常（如无标点）|不影响工具使用，答题时按用户输入的内容展示，用户自行核对答案。|
|漏复习多日，next时间远早于当前日期|自动将next时间顺延至当前日期，用户打开工具后，该诗词显示在今日待复习列表中，无需手动调整。|
|切换用户后，未完成的答题弹窗|切换用户时，自动关闭答题弹窗，不记录本次复习，避免数据混乱。|
