## 修复 Kokoro TTS Batch 行数不匹配错误
- 错误：`[Kokoro Batch] Error generating chunk: number of lines in input and output must be equal, we have: input=3, output=4`
- 原因：`split_text()` 按句子边界切分但不处理换行符，导致 chunk 内嵌换行；Kokoro 内部按换行切分输入，音素化产出行数与输入不一致时报错
- 修复1：重写 `split_text()`，先按换行符拆分确保每个 chunk 是单行，再在每行内按句子边界切分
- 修复2：增强 `kokoro_tts_batch()` 容错——单个 chunk 失败时尝试逐行重试，仍失败则插入 0.3s 静音占位而非让整个请求 503

## 修复 Kokoro TTS Batch 并行导致吞字问题
- 症状：诗歌等多行文本 TTS 朗读时，某一行（如 "In a beautiful pea-green boat,"）被吞掉，高亮正常
- 根因：`kokoro_tts_batch()` 使用 `ThreadPoolExecutor` 并行调用 `kokoro.create()`，但 Kokoro ONNX 推理不是线程安全的，并行调用会导致部分 chunk 产出空/损坏音频（不报错但数据无效）
- 修复：将并行改为顺序生成 + 加锁（`_kokoro_gen_lock`），同时 `kokoro_tts()` 单次调用也加锁
- 附加优化：chunk 间插入 50ms 静音模拟自然断句；音频有效性校验（空检测 + 时长/字符比例检测）；采样率不一致时线性插值重采样；失败重试 2 次后静音占位
