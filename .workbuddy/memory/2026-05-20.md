## 修复查看原文 answer-canvas 渲染无文字
- 症状：点击"查看原文"后 canvas 区域无文字内容
- 根因1：`renderAnswerCanvas()` 没有 return 值，`revealAnswer()` 中 `canvas` 变量为 undefined，`canvas.style.display = 'block'` 抛 TypeError，导致 canvas 始终 display:none
- 根因2：`article.text` 含 `\n` 换行符，canvas fillText 不处理 `\n`，`\s+` split 后 `\n` 作为不可见 word 渲染
- 修复1：`renderAnswerCanvas()` 末尾加 `return canvasEl`
- 修复2：渲染前将 `\n` 替换为双空格 `flatText = text.replace(/\n/g, '  ')`
- 修复3：`revealAnswer()` 用 try-catch 包裹 canvas 渲染，失败时回退到纯文本模式（显示 reveal-text）

## 修复弹窗关闭后上一个音频仍播放的竞态问题
- 症状：打开弹窗自动播放，如果文本较长未缓存，请求未完成时关闭弹窗，再打开另一个弹窗会播放上一个音频
- 根因：`toggle()` 中 `await ttsFetch()` 是异步操作，关闭弹窗时 `cleanup()` 暂停了当前 audio，但 `await` 返回后又 `new Audio(url).play()` 重新播放
- 修复1：`AudioPlayer` 和 `PronunciationPlayer` 增加 `_requestId` 机制——每次 `cleanup()` 和 `toggle()` 递增，`await` 返回后检查 `requestId` 一致性，不一致则放弃播放
- 修复2：`openListenModal()`/`renderBatchQuestion()` 的 `setTimeout` 改为可取消（`_singleAutoPlayTimer`/`_batchAutoPlayTimer`），`closeListenModal()`/`closeBatchModal()` 中 `clearTimeout`
- 修复3：`ttsFetch()`/`fetchTtsWithTimestamp()` 增加 `abortSignal` 参数，`AudioPlayer`/`PronunciationPlayer` 增加 `_abortCtrl`（AbortController），`cleanup()` 时 `abort()` 中断网络请求，避免浪费带宽
- 修复4：`switchMode('pronunciation')` 时也 `clearTimeout` 两个自动播放定时器

## Bug修复：rewards[rewardKey]未写入
- **症状**：同一道题重复练习点击"加入复习计划"后积分持续增加，dailyLog[today].rewards[rewardKey]值未持久化
- **根因**：`joinReviewPlan()`和`completeListening()`中，先在用户对象A上设置`rewards[rewardKey]=true`，然后`u=recordProgress(...)`内部重新`safeUser(getUser())`返回全新对象B覆盖了u，A上的rewards标记丢失，最终`setUser(u)`保存的是B（不含rewards标记）
- **修复**：将rewards检查和写入移到`recordProgress()`调用之后，确保写入的是最终被`setUser(u)`保存的对象
